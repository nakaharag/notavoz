# speech-to-text.me - Style Guide Generation Prompt

Use this prompt with AI design tools (Midjourney, DALL-E, Figma AI, or design assistants) to generate a visual style guide.

---

## Prompt for Style Guide Image

```
Create a modern web application style guide for "speech-to-text.me" - a voice transcription and AI summarization tool.

Brand personality: Professional yet approachable, tech-forward, clean, trustworthy

Requirements:
1. Color palette (5-6 colors):
   - Primary color (for buttons, accents)
   - Secondary color (for highlights)
   - Background colors (light/dark modes)
   - Text colors (primary, secondary, muted)
   - Success/error states

2. Typography:
   - Heading font (modern sans-serif)
   - Body font (readable, clean)
   - Monospace font (for transcripts)

3. UI Components:
   - Large record button (microphone icon)
   - Text boxes for transcript and summary
   - Share buttons (copy, link, social icons)
   - Ad banner placeholders
   - Loading/processing states

4. Layout:
   - Single-page centered layout
   - Mobile-responsive
   - Clear visual hierarchy
   - Generous whitespace

Style direction:
- Modern and minimal
- Slight gradient accents
- Soft shadows
- Rounded corners
- Dark mode option

Show: Color swatches with hex codes, typography samples, button states, component examples, and a mockup of the main interface.
```

## Prompt for Logo

```
Design a minimal, modern logo for "speech-to-text.me"

Concept: Voice/audio transforming into text

Elements to consider:
- Sound wave
- Microphone
- Text/document
- Transformation/conversion arrow

Style:
- Minimal line art or geometric
- Works in single color
- Scalable (favicon to large)
- Modern tech aesthetic

Colors: Use the brand primary color

Variations needed:
1. Full logo with wordmark
2. Icon only (for favicon)
3. Horizontal layout
4. Dark background version
```

```
Create a complete UI kit for speech-to-text.me including:

1. Atoms:
   - Buttons (primary, secondary, ghost, icon-only)
   - Input fields
   - Text styles (h1-h6, body, caption)
   - Icons (mic, copy, share, twitter, facebook, whatsapp)

2. Molecules:
   - Record button with states (idle, recording, processing)
   - Text box with copy button
   - Share button group
   - Ad placeholder banner

3. Organisms:
   - Header with logo
   - Main recording section
   - Transcript display card
   - Summary display card
   - Share panel
   - Footer

4. Templates:
   - Landing page (before recording)
   - Recording in progress
   - Results page (transcript + summary)
   - Shared content view (/s/[id])

5. States:
   - Empty state
   - Loading/processing
   - Success
   - Error
   - Rate limit reached
```

## Brand Voice

**Tagline options:**
- "Record. Transcribe. Share."
- "Your voice, in text, instantly."
- "Speak it. Read it. Share it."
- "Voice to text, powered by AI."

**Tone:**
- Clear and direct
- Helpful, not salesy
- Tech-savvy but accessible
- Brief microcopy

**Example UI text:**
- Button: "Start Recording"
- Processing: "Transcribing your audio..."
- Success: "Transcription complete!"
- Share: "Share this transcription"
- Error: "Oops! Something went wrong. Try again."
- Rate limit: "Daily limit reached. Come back tomorrow!"
